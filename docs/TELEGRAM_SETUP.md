# 📱 Получение Telegram Chat ID

## Метод 1: Через API Telegram (Рекомендуется)

### Шаг 1: Найти вашего бота
1. Откройте Telegram
2. В поиске введите: `@SalesBotMVP_bot` (или имя вашего бота)
3. Нажмите **Start** или отправьте `/start`

### Шаг 2: Получить обновления
Выполните эту команду на сервере:

```bash
curl "https://api.telegram.org/bot7606566977:/getUpdates"
```

### Шаг 3: Найти Chat ID в ответе
Ответ будет выглядеть примерно так:
```json
{
  "ok": true,
  "result": [
    {
      "update_id": 123456789,
      "message": {
        "message_id": 1,
        "from": {
          "id": 987654321,
          "is_bot": false,
          "first_name": "Иван",
          "username": "ivan_user"
        },
        "chat": {
          "id": 987654321,    ⬅️ ЭТО ВАШ CHAT_ID!
          "first_name": "Иван",
          "username": "ivan_user",
          "type": "private"
        },
        "date": 1234567890,
        "text": "/start"
      }
    }
  ]
}
```

**Ваш Chat ID** = значение `"chat": {"id": 987654321}`

### Шаг 4: Добавить в .env
```bash
cd /root/salesbot-mvp
nano .env

# Найдите строку:
TELEGRAM_ADMIN_CHAT_IDS=[]

# Замените на (вставьте свой chat_id):
TELEGRAM_ADMIN_CHAT_IDS=["987654321"]

# Для нескольких админов:
TELEGRAM_ADMIN_CHAT_IDS=["987654321", "123456789", "555666777"]

# Сохранить: Ctrl+O, Enter, Ctrl+X
```

### Шаг 5: Перезапустить сервис
```bash
sudo systemctl restart salesbot-api
```

### Шаг 6: Протестировать
```bash
cd /root/salesbot-mvp
python3 << 'EOF'
import asyncio
from app.alerts.telegram_alerts import telegram_alerts

async def test():
    # Замените 987654321 на ваш chat_id
    result = await telegram_alerts.send_telegram_message(
        "987654321",
        "✅ <b>Тест успешен!</b>\n\n🤖 SalesBot подключен и работает!\n\n📊 Теперь вы будете получать алерты о:\n• Просроченных обещаниях\n• Необработанных лидах\n• Ежедневную сводку"
    )
    print(f"\nРезультат: {'✅ УСПЕХ - Проверьте Telegram!' if result else '❌ ОШИБКА'}")

asyncio.run(test())
EOF
```

## ❓ Troubleshooting

### Проблема: getUpdates возвращает пустой массив
**Решение:**
1. Убедитесь что вы отправили `/start` боту
2. Подождите 1-2 секунды
3. Повторите запрос getUpdates

### Проблема: Тест отправки не работает
**Решение:**
```bash
# Проверьте что chat_id в .env без кавычек внутри списка:
cat /root/salesbot-mvp/.env | grep TELEGRAM_ADMIN

# Должно быть:
# TELEGRAM_ADMIN_CHAT_IDS=["123456789"]

# НЕ должно быть:
# TELEGRAM_ADMIN_CHAT_IDS='["123456789"]'
```

### Проблема: Chat ID для группы
Если хотите отправлять в группу Telegram:

1. Добавьте бота в группу
2. Сделайте бота администратором
3. Отправьте что-то в группу
4. Выполните getUpdates
5. Chat ID группы будет отрицательным (например: `-1001234567890`)
6. Используйте его с минусом:
   ```bash
   TELEGRAM_ADMIN_CHAT_IDS=["-1001234567890"]
   ```
